Uso de modelos de IA:

Se utilizó la ayuda de modelos de IA como chatgpt para entender y visualizar las necesidades de este proyecto, por ejemplo:

El uso de IntelliJ, un IDE muy completo para Java y que ofrece herramientas para el FXML, se contempló que IDE sería mejor usar y la IA sugirió IntelliJ.

La arquitectura MVC, se le preguntó a la IA como iniciar el proyecto para su uso, utilizando formato Maven para facilitar el uso de librerías externas (JavaFX).
Además, se le pidió hacer un boceto para contemplar cuantos diferentes paquetes (carpetas) crear.

Se le pidió a la IA resolver errores en dependencias y crear lineas necesarias para que el module-info.java permitiera ejecutar el proyecto en JavaFX.

Se le pidió asistencia a la IA para acomodar los diferentes botones ubicados en los FXML, (cuestión de simple diseño ya que es dificil acomodar pixel por pixel).

Para la creación de cuestionarios en FXML se le pidió asistencia al crear las validaciones y sus typeados (ya que Java es muy estricto), ejemplo: txtPuntosActuales.setText(String.valueOf(cliente.getPuntos()));

Se solicitó ayuda a la IA para crear tablas con diseño básico para ordenar objetos listados.

Se le pidió ayuda a la IA para "generalizar" métodos de lectura de los archivos que se guardan y cargan.

Se le solicitó apoyo a la IA para importar todas las librerias y archivos necesarios para cada archivo nuevo creado.

Se le solicitó ayuda a la IA para cambiar en muchas ocaciones metodos del MainApp ya que con cada nueva mecanica o utilidad se tenia que organizar casi todos los FXML para un solo boton.

Se le solicitó apoyo a la IA para cambiar ocurrencia de clases en el FXML para el css.

Se usó la IA para crear la documentación del código bajo supervisión de un humano.
